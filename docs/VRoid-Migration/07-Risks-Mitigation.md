# Risks and Mitigation

## 主要リスク

1. 非VRoid VRM の回帰
- 影響: 既存ユーザーへの破壊的影響
- 緩和策: 全PRで非VRoid既存成功ケース回帰を必須化
- 監視: 主要メトリクス差分（ボーン/メッシュ/モーフ）

2. 自動判定の誤判定
- 影響: 不適切なprofile選択による失敗
- 緩和策: 判定理由ログを保持し、判定順序を固定
- 監視: 判定結果分布と失敗率

3. 共通化による副作用
- 影響: generic経路の品質劣化
- 緩和策: 共通化後すぐに回帰ゲートを設定
- 監視: PR-B時点の回帰結果

4. VRoid固有処理の漏れ
- 影響: 変換完走率低下
- 緩和策: ボーン/メッシュ/モーフ/物理の機能単位移植
- 監視: サンプル別完走率

5. frontend案内不足
- 影響: ユーザーが失敗理由を判断できない
- 緩和策: 判定不能/変換失敗の案内文を標準化
- 監視: 手動スモークで再試行可能性を確認

## 診断ログ方針

- 最低限ログ項目
  - 判定入力キー
  - 判定結果
  - 選択profile
  - 失敗ステージ
- 個人情報や不要な生データは出力しない。

## ロールバック条件

- 非VRoid回帰で1件でも致命的不合格が出た場合
- 判定誤りで変換失敗率が閾値超過した場合
- マイルストーンゲート未達の場合

## ロールバック手順（概要）

1. 直近PR単位で戻す
2. 回帰基準を再実行
3. 原因が特定できる最小差分に分割して再投入
