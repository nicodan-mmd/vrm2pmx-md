# Test Strategy

## テスト目的

- 非VRoid既存変換の互換維持を最優先で保証する。
- VRM1.0 / VRoid 追加経路の安定性を確認する。

## テストレイヤー

1. Unit
- 判定レイヤーの判定結果
- 共通化ユーティリティの入出力
- Profileごとの単機能

2. Contract
- Detection Layer の返却契約
- frontend が受け取る表示メッセージ契約

3. Integration
- 自動判定 -> profile選択 -> 変換完走
- 失敗時メッセージの連携

4. Regression
- 非VRoid VRM（既存成功ケース）
- VRM1.0 サンプル
- VRoid サンプル

## 判定指標

1. PMX出力有無
2. ボーン数
3. メッシュ数
4. 主要モーフ数
5. 致命的例外の有無

## 判定ルール（詳細）

1. Hard Fail（即停止・要確認）
- PMXが生成できない
- PMX読み込み時に例外が発生する
- 頂点インデックス不整合、NaN/Inf座標などモデル破綻が発生する
- 非VRoid既存成功ケースが変換不能になる

2. Soft Warn（表示優先で継続）
- ボーンが欠損していても、モデル表示が成立する場合は継続する
- 表示は成立するが、ボーン数やモーフ数が基準値から乖離する
- マテリアルや表情の一部欠落など、致命的でない差分がある
- この場合はエラーレポートへ記録し、次PRで改善対象に積む

3. 初期しきい値（Phase 0確定案）
- ボーン数: baseline比 ±3% 以内を目安
- メッシュ数: baseline一致を原則（差分が出たら要確認）
- 主要モーフ数: baseline比 ±10% 以内を目安
- 注記: しきい値超過でも Hard Fail 条件を満たさなければ一旦進める

## 合格条件

- 非VRoid VRMの既存成功ケースで、少なくとも PMX生成と表示成立を満たす。
- VRM1.0 / VRoid サンプルで変換が完走する。
- 判定不能時に定義済み案内が表示される。
- 表示成立モデルは Soft Warn として継続可能である（エラーレポート記録を前提）。

## 推奨実行タイミング

- PR-B 以降: 非VRoid回帰を毎回実施
- PR-E 以降: VRM1.0 / VRoid 統合回帰を毎回実施
- マージ前: 全サンプルフル回帰

## テスト成果物

- サンプル別結果表
- 失敗ケース再現手順
- 回帰比較ログ

## サンプル運用（2026-03-25時点）

- 非VRoidは `CosmicPerson.vrm`、`UnityChan.vrm`、`Zundamon_2025_VRM09A.vrm` を採用。
- VRM1.0 は `AvatarSample_A.vrm`（v1.0）と `Zundamon_2025_VRM10A.vrm` を採用（`AvatarSample_A_1.0.vrm` は予備）。
- VRoid は `プロレスラー_リンリン.vrm` と `4590875026920364234.vrm` を採用。
- 入力サンプルは外部パス管理のため、実行ログに「使用ファイル絶対パス」を必ず記録する。
- サンプル一覧の正本は docs/VRoid-Migration/sample-set-20260325.json とする。
