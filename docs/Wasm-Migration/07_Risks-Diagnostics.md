# Risks and Diagnostics

## 主要リスク

1. 初回ロード遅延
- Pyodide と依存ロードで初回待ち時間が長くなる。

2. メモリ使用量
- 大きい VRM でブラウザメモリを圧迫する可能性がある。

3. 既存依存との非互換
- GUI 前提コードが残るとブラウザ実行に失敗する。

4. 変換品質差分
- I/O 経路変更により出力差が発生する可能性がある。

## 監視項目

- ロード完了までの時間
- 変換完了までの時間
- 失敗率（入力不正/実行例外/メモリ不足）
- 主要サンプルの出力差分件数

## 診断ログ方針

- Worker 側で段階ログを構造化して出す。
- 本番 UI には詳細トレースを直接表示しない。
- ユーザー向けは簡潔なコード付きメッセージを返す。

## ロールバック方針

- Wasm 経路が不安定な期間は、開発用途として FastAPI 経路を残す。
- 機能フラグで実行経路を切り替え可能にする。
- 品質基準未達の場合は公開ブランチへの反映を停止する。
- Wasm 経路が安定した時点で、FastAPI 経路は必須から任意に切り替える。

## 先行確認チェックリスト

- [ ] GUI 依存 import の解消方針が確定している
- [ ] `bytes -> bytes` API の仕様が確定している
- [ ] Pyodide 初期化と Worker 実行の責務分離が確定している
- [ ] 最低限の回帰サンプルが確定している
