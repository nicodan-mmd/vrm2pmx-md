# Go版追随メモ（Python版との差分整理）

## このドキュメントの目的

この文書は、Go版 `mu_vrm2pmx` と本プロジェクト（Pythonベース）を比較し、
「Go版で追加・強化されているロジック」と「追随候補」を整理するためのメモです。

## 差分サマリ（3列）

| 項目                              | Go版                                                                                | Python版（現状）                                               |
| --------------------------------- | ----------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| 出力レイアウト準備                | `tex` / `glTF` ディレクトリ作成、補助アーティファクト出力、特殊目テクスチャ展開あり | 変換本体（VRM読込→PMX出力）中心                                |
| 特殊目テクスチャ（eye_star など） | 組み込みアセットを出力先へ展開する専用処理あり                                      | 同等の「埋め込み展開」専用処理は未確認                         |
| 材質バリアント正規化              | 表面/裏面/エッジへの複製・接尾辞正規化・命名整理あり                                | `alphaMode` ベースの並びはあるが、同等のバリアント複製は未確認 |
| 透明材質の並べ替え                | UVサンプリング、テクスチャalpha評価、奥行き推定を使った並び替え                     | OPAQUE/MASK/BLEND などの分類中心                               |
| ボーンマッピング強化              | 不足ボーン追加、命名整理、表示枠再構成、舌ボーン系列再構成あり                      | 対応表ベースの変換中心                                         |

## 追随の優先候補（提案）

1. 材質並べ替えロジックの強化

- 見た目破綻に直結しやすく、効果が分かりやすい。

2. 出力レイアウト（tex/glTF/特殊目アセット）整備

- 再現性と配布品質を上げやすい。

3. ボーンマッピング拡張（舌ボーン含む）

- モデル互換性と表情品質の改善に効く。

4. モーフ rename-only 拡張

- VRM差異吸収の即効性が高い。

5. Aスタンス補正の明示化

- 入力モデル姿勢の揺れに対する安定化。

## ライセンス整理（重要）

### 結論（実務向け要約）

- Go版ソースは **CC-BY-NC-4.0** なので、**非商用なら利用・改変・取り込みは原則可能** です。
- ただし、取り込み時は CC-BY-NC-4.0 の条件（帰属表示、改変明示、ライセンス表示など）を満たす必要があります。
- また、Go版の `LICENSE-EXCEPTION` は「著作権者本人が配布する公式バイナリ」の商用利用例外であり、
  **第三者がソースを取り込んで作る派生物の商用利用を許可するものではありません**。

### 注意点（MITプロジェクトへの混在）

- 本プロジェクトのライセンスは MIT ですが、CC-BY-NC-4.0 のコードを取り込むと、
  その取り込み部分（および派生部分）には非商用制約がかかります。
- そのため、配布時に「全体をMITのみ」と表現すると不正確になる可能性があります。
- 実務上は、少なくとも以下を整備するのが安全です。

1. 取り込み元ファイル・コミット・著作者を明記する（帰属）。
2. 改変した箇所を明示する（変更履歴/注記）。
3. CC-BY-NC-4.0 対象部分を分離し、ライセンス境界を明示する。
4. 配布物のライセンス表記を複数ライセンス構成として整理する。

### 「商用じゃなければ問題ない？」への回答

- **概ね Yes（非商用利用で条件順守なら可）**。
- ただし「何が商用に当たるか」「配布形態で条件を満たしているか」は事案依存です。
- 最終判断が必要な場合は、必ず権利者確認または法務確認を推奨します。

## 方針追記（2026-03-27）

### プロダクト方向性

- 想定ユーザーは、モデラーガチ勢よりも「VRoidをMMDで踊らせたい」層を主対象とする。
- そのため、最優先は「Webで簡単に変換できる体験（導線の短さ、失敗しにくさ、分かりやすさ）」とする。

### Go版の位置づけ

- Go版は品質向上の観点で学ぶ価値がある（特に材質・透明・ボーン/モーフ補正）。
- ただし本プロジェクトでは、Go版を全面採用するより、必要な考え方を参照する位置づけを基本とする。

### 実装戦略メモ

- 既存の Python + Wasm 経路は維持し、まずは Web 体験を改善する。
- 変換コアの将来的な Rust 化は選択肢として有望。
- 進め方は段階的に行い、UI/UX を先に良くしつつ、変換品質は比較検証しながら取り込む。

### 優先順位（当面）

1. Webでの使いやすさ改善（アップロード、進捗表示、失敗時の案内、出力導線）。
2. Python版の安定運用と回帰防止。
3. Go版の有効ロジックを仕様として整理（直接移植より先に仕様化）。
4. 小さな単位で Rust 実験（まずは1機能を比較実装）。
